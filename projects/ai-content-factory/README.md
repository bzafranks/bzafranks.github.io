# AI Educational Content Factory
### An Enterprise AI Platform for Automated Curriculum & Learning-Content Production

Project #14 in the **Curriculum Innovation & Instructional Design Portfolio** of Dr. Barbara Z.
Franks, part of the **AI for Education Innovation Suite**. Built on the portfolio's shared design
system; deployed at **https://bzafranks.github.io/projects/ai-content-factory/**.

## 📘 Project Overview
An enterprise content-production platform that turns a **single prompt into a complete instructional
package** — lesson, workbook, slides, video, assessment, rubric, quiz/game, interactive activity, and
teacher/student/parent guides, plus an LMS package and publishing files. Not an AI lesson generator: an
orchestrated production line with a human editorial gate.

## 🏭 Educational Publishing Challenge
Organizations spend enormous time producing materials and recreating the same content in multiple
formats. The Factory automates the production pipeline while editors guarantee quality.

## 🧠 AI Content Factory Philosophy
**AI produces at scale; editors guarantee quality.** Every asset is a draft that passes a human
editorial gate (pedagogy, bias, accessibility, copyright, alignment) before publishing.

## ✨ Platform Features
- **Content Factory** — one prompt (grade, subject, topic, model, language, format) → a 13-asset package
- **Content Wizard** — guided creation with standards, model, accessibility, and branding
- **Generators** — lesson, assessment, AI video, interactive, presentation, workbook, teacher guide,
  parent resources
- **Workflow Automation** — visual production pipeline that fans one input out to a full suite
- **LMS Package Builder** — Canvas, Google Classroom, Moodle, Blackboard, Schoology, Brightspace, SCORM
- **Publishing Center** — website, PDF, PPT, Word, eBook, online course, mobile, print kit
- **Content Library** — searchable, filterable, version-controlled store
- **Analytics** — production metrics, time saved, assets produced, quality pass rate
- **Enterprise Admin** — users, permissions, teams, approvals, version history, brand management
- **Settings & QA** — responsible-AI & quality-assurance framework, implementation guide
- **Product UX:** command palette (Ctrl/⌘ + K), FAB, autosave, dark mode, accessibility toolbar

## 🛠️ Technology Stack
Shared portfolio **design-system.css** + **portfolio.js**, plus project-local **`style.css`**
(enterprise chrome: production pipeline, content-package grid, wizard, export grid, dark mode) and
**`script.js`** (engine: the single-prompt package factory, per-asset generators, animated workflow
pipeline, publishing/LMS wizard, library, analytics). Vanilla HTML5/CSS/JS, `localStorage`, no build
step. **All AI is simulated client-side; sample content is illustrative and requires editorial review.**

## ▶ Run Locally
