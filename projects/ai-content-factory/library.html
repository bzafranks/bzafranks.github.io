Version control

Every item in the library carries a full version history, so nothing is ever lost and every change is traceable:

  • Drafts — new or in-progress content that is editable and not yet live. Each save creates a restorable revision.
  • In review — drafts submitted to the editorial gate for pedagogical and accessibility review before approval.
  • Published — approved, locked versions distributed to LMS, web, or print. Republishing supersedes the prior published version.
  • Archived — retired versions kept for audit and rollback; restorable to a draft at any time.

Versions are numbered (v1, v2, …); you can compare revisions, roll back to any prior state, and see who changed what and when.
