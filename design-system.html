Guidelines

Building New Projects

  • Start from a copy of templates/project-template.html and lesson-template.html.
  • Link the shared design-system.css and portfolio.js — never fork them.
  • Set data-project and data-project-root on <body> for the portfolio ribbon.
  • Use existing components and tokens; if a new component is truly needed, add it to the shared CSS so all projects gain it.
  • Add one entry to assets/js/projects-data.js to publish the project on the homepage.
  • Assign a pillar: Instructional Design or Instructional Leadership.
  • Maintain accessibility: keyboard support, captions, color-plus-text, printable fallbacks.

Full written guidance lives in STYLE_GUIDE.md at the repo root.
